# Go2Z1 精简实验大纲
---

## 一、固定基础设施（Baseline Codebase 技术锚点）

所有实验共享以下已高度优化的底层组件，视为**不可消融的固定模块**，保证各组控制变量对齐：

| 模块               | 技术实现                                                          | 在实验中的角色                      |
| ------------------ | ----------------------------------------------------------------- | ----------------------------------- |
| **安全约束优化**   | PPO-Lagrangian CMDP，双 Critic + PID 双重上升学习 Lagrangian 乘子 | 所有 RL 实验的安全层统一基准        |
| **层次化运动控制** | 零拷贝 JIT 编译低级运动策略，3D 速度指令 → 12-DOF 关节目标        | 创新点 1/2 的底层执行器，不参与消融 |
| **负载模拟**       | 物理 substep 内运动学"焊接"（Kinematic Welding）                  | 所有挂载 Z1 实验的统一负载仿真方案  |
| **抓取课程**       | 动态空间阈值线性衰减 Curriculum                                   | 创新点 3 抓取实验的起点课程设计     |
| **场景配置**       | 直接 SdfLayer USD 操作绕过 Omniverse Session Layer                | 所有 IsaacLab 场景生成的统一方案    |
| **观测归一化**     | 纯 GPU 驻留 Welford 在线算法                                      | 消除主机-设备瓶颈，全组统一使用     |

## 二、创新点 1：采样算法（实时性核心）

**核心问题：** 什么采样方法能在 Go2 算力约束下，以最低延迟维持可接受的避障质量？

只做一条主线，砍掉表征空间和混合架构两个主轴，聚焦于**采样速度 × 安全引导**的组合：

| 实验组  | 方法                            | 定位                                                                        |
| ------- | ------------------------------- | --------------------------------------------------------------------------- |
| **B0**  | DDPM 50步                       | 基线                                                                        |
| **E-1** | Flow Matching（8步）            | 速度对比                                                                    |
| **E-2** | Consistency Policy 1步（CP-CD） | 速度上限                                                                    |
| **E-3** | **E-2 + SDF Energy Guidance**   | 核心创新：1步生成 + 采样时注入碰撞惩罚梯度，无需重训练即实现 Zero-shot 避障 |

**E-3 是主推创新**，E-1/E-2 作为铺垫对比，B0 作为锚点。共 4 组，结构清晰，故事线完整。

**评估指标：** 推理延迟、控制频率、静态/动态避障成功率、Orin 显存占用。

## 三、创新点 2：Sim-to-Real（迁移鲁棒性核心）

**核心问题：** 如何让仿真训练的策略零样本迁移到挂载 Z1 的真机上？

聚焦**两个最有把握出结果的方向**，砍掉 PAIRED/DreamerV3 对抗生成（工程代价极高）：

**主轴 A：带臂动态的 Teacher-Student**

| 实验组  | 方法                                                                                                     | 定位         |
| ------- | -------------------------------------------------------------------------------------------------------- | ------------ |
| **B0**  | 端到端直接训练，本体感觉 + 含噪视觉                                                                      | 基线         |
| **E-1** | 标准 RMA（教师用特权地形信息，学生用本体感觉）                                                           | 已有方法对比 |
| **E-2** | **E-1 + Z1 动量/重心编码**：教师额外观测机械臂实时惯性；学生用轻量 LSTM 从腿部受力预测负载，提前补偿步态 | 核心创新     |

**主轴 B：硬件感知延迟建模**（可与主轴 A 正交叠加，作为 E-2 的强化版）

| 实验组          | 方法                                                                                                                   |
| --------------- | ---------------------------------------------------------------------------------------------------------------------- |
| **B0**          | 仿真零延迟                                                                                                             |
| **E-3**         | 固定 50ms 视觉延迟 + 2ms 本体感觉延迟                                                                                  |
| **E-4（创新）** | **随机延迟区间（50–100ms）+ Temporal Offset 位置编码**：Transformer 显式感知视觉帧的「老旧程度」，而非盲目融合错位数据 |

两个主轴共 5–6 组实验，可在仿真阶段全部完成，真机只需验证最优组。

**评估指标：** Sim-to-Real Gap（成功率差值）、机械臂甩动干扰下倾覆率、平衡恢复时间。

## 四、创新点 3：VLA 抓取（泛化性核心）

**核心问题：** 如何在有限算力下抓取训练集外的新物体？

砍掉 RAG、渐进蒸馏等工程量大的方向，聚焦**一条端云协同主线**：

| 实验组              | 方法                                                                       | 定位                                                                                    |
| ------------------- | -------------------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| **B0**              | 云端完整 VLA + Wi-Fi 下发动作（高延迟，断网即崩）                          | 基线上限                                                                                |
| **E-1**             | 端侧动作分布蒸馏小模型（~200M），Orin 独立推理                             | 无网络对比                                                                              |
| **E-2**             | **云端输出可供性热力图 + 语义关键点 → 端侧轻量 Policy Head 50Hz 高频控制** | 核心创新：语义推理（1Hz 云端）与动作控制（50Hz 端侧）彻底解耦，断网后端侧可独立降级运行 |
| **E-3**（可选扩展） | DAgger 在线蒸馏：真机部署时云端教师持续修正 E-1 的端侧模型                 | 若 E-1 精度不足时的补救路线                                                             |

**E-2 是主推创新**，强调「语义与控制频率解耦」这一核心 insight。E-3 视 E-1 效果决定是否执行。

**评估指标：** 训练集物体抓取成功率、新物体 Zero-shot 成功率、断网状态下成功率、端侧推理延迟。

## 五、实验优先级与执行顺序

```
Phase 1（仿真，最快出结果）
  └─ 创新点1：B0 / E-1 / E-2 / E-3 全部在 IsaacLab 内完成
  └─ 创新点2 主轴B：延迟建模 E-3 / E-4（纯仿真改动，低成本）

Phase 2（仿真，中等工程量）
  └─ 创新点2 主轴A：Teacher-Student E-1 / E-2（需设计 LSTM 负载编码器）

Phase 3（系统集成，最重）
  └─ 创新点3：E-1 蒸馏 → E-2 端云协同联调 → 真机验证
  └─ 创新点1 E-3 真机部署验证（SDF Guidance 是否迁移）
```

## 六、统一评估指标（精简版）

| 维度     | 指标                           |
| -------- | ------------------------------ |
| 实时性   | 推理延迟（ms）、控制频率（Hz） |
| 安全性   | 避障成功率、碰撞次数           |
| 灵巧性   | 抓取成功率（训练集 / 新物体）  |
| 鲁棒性   | 断网成功率、机械臂干扰下倾覆率 |
| 算力效率 | Orin 显存占用（GB）            |

---

**核心故事线总结：** 三个创新点各有一个「1步生成+SDF引导」、「带臂动态 Teacher-Student」、「端云频率解耦」的主推创新，其余实验组均服务于烘托和对比，总实验组数控制在 **15组以内**，真机验证只需跑最优组合。
