# Go2Z1 实验文档大纲

## 一、系统总览与研究目标

**研究目标：** 在 Unitree Go2 + Z1 机械臂组合平台上，以 IsaacLab 5.1 为仿真环境、外挂算力为推理支持，实现具备实时性、安全性与开放词汇泛化能力的全身协调移动操作（Whole-Body Loco-Manipulation）。

**三大创新方向：**

- **创新点 1**：扩散 / Flow Matching / 加速采样算法的对比与改进（实时性与算力适配）
- **创新点 2**：虚拟环境 Sim-to-Real 训练方法的改进（迁移鲁棒性）
- **创新点 3**：VLA 识别抓取方法的端云协同设计（开放词汇泛化）

---

## 二、固定基础设施（Baseline Codebase 技术锚点）

所有实验共享以下已高度优化的底层组件，视为**不可消融的固定模块**，保证各组控制变量对齐：

| 模块               | 技术实现                                                          | 在实验中的角色                      |
| ------------------ | ----------------------------------------------------------------- | ----------------------------------- |
| **安全约束优化**   | PPO-Lagrangian CMDP，双 Critic + PID 双重上升学习 Lagrangian 乘子 | 所有 RL 实验的安全层统一基准        |
| **层次化运动控制** | 零拷贝 JIT 编译低级运动策略，3D 速度指令 → 12-DOF 关节目标        | 创新点 1/2 的底层执行器，不参与消融 |
| **负载模拟**       | 物理 substep 内运动学"焊接"（Kinematic Welding）                  | 所有挂载 Z1 实验的统一负载仿真方案  |
| **抓取课程**       | 动态空间阈值线性衰减 Curriculum                                   | 创新点 3 抓取实验的起点课程设计     |
| **场景配置**       | 直接 SdfLayer USD 操作绕过 Omniverse Session Layer                | 所有 IsaacLab 场景生成的统一方案    |
| **观测归一化**     | 纯 GPU 驻留 Welford 在线算法                                      | 消除主机-设备瓶颈，全组统一使用     |

## 三、创新点 1：采样算法对比实验

### 1.1 主轴 A：采样速度 × 质量 Pareto 前沿

**研究问题：** 在 Go2 Orin（有限算力）和外挂算力两种部署场景下，哪类采样算法能以最低延迟达到可接受的轨迹质量？

| 实验组         | 方法                                   | 步数    | 预期延迟 |
| -------------- | -------------------------------------- | ------- | -------- |
| **B0（基线）** | 标准 DDPM                              | 50 步   | ~200ms   |
| **B1**         | DDIM                                   | 20 步   | ~80ms    |
| **E-A**        | DPM-Solver++                           | 5–10 步 | ~30ms    |
| **E-B**        | Flow Matching / Rectified Flow         | 5–8 步  | ~25ms    |
| **E-C**        | Consistency Policy（CP-CD，蒸馏版）    | 1–2 步  | ~5ms     |
| **E-D**        | Consistency Policy（CP-CT，从头训）    | 1–2 步  | ~5ms     |
| **E-E**        | iConsistency Models（2024）            | 2–3 步  | ~8ms     |
| **E-F**        | Shortcut Models（Flow 领域一致性类比） | 1 步    | ~3ms     |

**评估指标：** 推理延迟（ms）、控制频率（Hz）、Orin GPU 显存占用、静态障碍物避障成功率、动态障碍物避障成功率。

---

### 1.2 主轴 B：表征空间选择（扩散生成 What？）

**研究问题：** Go2+Z1 的 18-DOF 异构动作空间应如何分解？

| 实验组  | 表征空间                              | 说明                                                                 |
| ------- | ------------------------------------- | -------------------------------------------------------------------- |
| **B0**  | 状态空间扩散（Diffuser 范式）         | 对未来状态轨迹扩散，底层 PD/MPC 追踪                                 |
| **E-A** | 动作空间扩散（Diffusion Policy 范式） | 关节角速度 / 力矩 Action Chunking                                    |
| **E-B** | **解耦双噪声调度**（本实验创新）      | 腿部用余弦调度，机械臂用 VP-SDE 线性调度；Cross-Attention 层交互对齐 |
| **E-C** | 自适应调度                            | 运行时根据 Z1 负载动态调整 β schedule（Hardware-Aware）              |

---

### 1.3 主轴 C：测试时引导（Safety Guidance）

**研究问题：** 如何在采样阶段注入物理安全约束，实现 Zero-shot 避障？

| 实验组         | 引导机制                                                                                                            |
| -------------- | ------------------------------------------------------------------------------------------------------------------- |
| **B0（基线）** | 无引导，纯端到端扩散策略                                                                                            |
| **E-A**        | Classifier-Free Guidance（CFG），深度/LiDAR 为条件，调节 scale                                                      |
| **E-B**        | **SDF Energy-Guided Sampling**（创新核心）：每步去噪同时加入 SDF 碰撞惩罚梯度，分数函数 = 数据梯度 + 远离障碍物梯度 |
| **E-C**        | DPPO：PPO 奖励信号在仿真中微调 Diffusion 的 Score Function，训练期天然注入安全偏好                                  |

---

### 1.4 主轴 D：扩散 + 经典优化混合架构

| 实验组  | 方法                                                                |
| ------- | ------------------------------------------------------------------- |
| **B0**  | 纯 Diffusion 生成最终动作                                           |
| **E-A** | Diffusion（3步）作为 CEM/MPPI Warm-start Seed，再用物理仿真迭代优化 |
| **E-B** | 扩散采样多峰轨迹 → 投票 / 集成选优                                  |

## 四、创新点 2：Sim-to-Real 训练方法

### 2.1 主轴 A：特权学习与师生蒸馏

| 实验组                        | 方法                                                                                                                      |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| **B0（基线）**                | 端到端，本体感觉 + 含噪视觉直接训练                                                                                       |
| **E-A（RMA 标准）**           | 两阶段 Teacher-Student，隐变量编码地形与物理参数                                                                          |
| **E-B（创新：带臂动态编码）** | 教师同时观测地形 + Z1 当前动量/重心偏移；学生用 Recurrent Belief Setter 从腿部受力 + 机械臂摆动阻力预测摩擦系数和负载质量 |

**与 Baseline Codebase 的交叉点：** 动态抓取 Curriculum 作为教师训练的阶段控制器，Kinematic Welding 提供教师可观测的负载真值。

---

### 2.2 主轴 B：域随机化策略

| 实验组          | 方法                                                                                  |
| --------------- | ------------------------------------------------------------------------------------- |
| **B0**          | 固定范围均匀分布 DR（标准基线）                                                       |
| **E-A**         | 自动课程学习 DR：随成功率提升动态扩大随机化范围（风力干扰、机械臂随机甩动幅度）       |
| **E-B（创新）** | 基于 PAIRED：对抗网络专门生成「刚好让当前策略失败」的场景                             |
| **E-C（创新）** | DR-Dreamer：用 DreamerV3 世界模型在 Latent Space 生成对抗场景，无需每次跑完整物理仿真 |

---

### 2.3 主轴 C：硬件感知延迟与噪声建模

| 实验组              | 建模细节                                                                                                                                     |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **B0**              | 视觉与电机信号完全同步、零延迟（仿真理想状态）                                                                                               |
| **E-A**             | 固定 50ms 视觉延迟 + 本体感觉 2ms 延迟                                                                                                       |
| **E-B**             | **随机延迟区间**（50–100ms 均匀采样），更接近真机波动                                                                                        |
| **E-C（创新核心）** | **延迟感知 Transformer**：在位置编码中引入时间戳差（Temporal Offset），让网络显式感知视觉帧的「老旧程度」；同时模拟运动模糊 + LiDAR 点云丢失 |

---

### 2.4 主轴 D：机械臂耦合动力学补偿

| 实验组          | 方法                                                              |
| --------------- | ----------------------------------------------------------------- |
| **B0**          | 不带补偿，Z1 视为死重                                             |
| **E-A**         | 质心动量控制（CMC）：将 Z1 动量纳入 Go2 底盘控制器状态空间        |
| **E-B（创新）** | 预测式补偿：轻量 LSTM 预测机械臂未来 100ms 动量变化，提前调整步态 |

**评估场景：** 机械臂随机甩动 + Go2 快速转向，测倾覆率与平衡恢复时间（Recovery Time）。

## 七、实验阶段规划

**Phase 0（环境与基线固化）：** IsaacLab 5.1 环境部署，PPO-Lagrangian CMDP + JIT 层次策略验证，Kinematic Welding 负载仿真验证，Welford GPU 归一化性能基准测试。

**Phase 1（创新点 1 - 仿真内）：** 采样算法对比（主轴 A），表征空间选择（主轴 B），SDF 引导采样实现（主轴 C），全在 IsaacLab 仿真内完成。

**Phase 2（创新点 2 - Sim2Real）：** 教师策略训练（带臂动态编码），对抗场景生成（PAIRED/DreamerV3），延迟感知 Transformer 训练，消融对比各域随机化方案。

**Phase 3（创新点 3 - VLA）：** 端侧蒸馏管线（特征蒸馏 → 动作分布蒸馏 → DAgger），物理动作向量数据库构建（RAG），云-端异步协同系统联调，可供性热力图生成与关键点下发验证。

**Phase 4（全系统集成与真机实验）：** 三大创新点联合部署，全因子场景测试，外挂算力 vs. Orin 独立性能对比，用户上传新物体 Zero-shot 抓取演示。

## 八、预期核心主张

1. 在有限算力约束下，**解耦双噪声调度 Consistency Policy + SDF Energy Guidance** 能以 ≤5ms 延迟实现媲美 50步 DDPM 的避障质量，首次将生成式轨迹策略推向 50Hz+ 实时控制。
2. **延迟感知 Temporal Offset Transformer + 带臂动态编码 Teacher-Student** 能将 Sim-to-Real Gap 控制在 X% 以内，实现 Zero-shot 真机部署。
3. **分层相似度（外观→可供性→物理属性）+ RAG 动作数据库** 能使系统对用户上传的任意新物体实现稳定抓取，无需重新训练。
